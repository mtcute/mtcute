# mtcute API Lookup

## Prefer This Order

1. Search the current app/repo for existing mtcute usage.
2. Use high-level `TelegramClient` methods and helper classes.
3. Use bundled lookup scripts for exact signatures and types.
4. Read local docs in `docs/guide`.
5. Use `https://ref.mtcute.dev/` or `https://mtcute.dev/llms.txt` for hosted docs when local docs are unavailable.
6. Use raw TL `.call()` only when no high-level method exists.

## Bundled Scripts

Resolve scripts relative to the `using-mtcute` skill directory, not the user's project. In examples below, replace `<skill-dir>` with the absolute path to the directory containing `SKILL.md`.

Run the command from the project or app root so package resolution uses that app's `node_modules` or mtcute workspace.

```bash
node <skill-dir>/scripts/get-method.js sendMedia
node <skill-dir>/scripts/get-method.js --search reaction
node <skill-dir>/scripts/get-method.js --list
```

`get-method.js` prints `TelegramClient` method JSDoc and signature.

```bash
node <skill-dir>/scripts/get-class.js Message
node <skill-dir>/scripts/get-class.js --search keyboard
node <skill-dir>/scripts/get-class.js --no-members Chat
```

`get-class.js` prints high-level exported class/interface/type definitions from `highlevel/types`.

```bash
node <skill-dir>/scripts/get-constructor.js user
node <skill-dir>/scripts/get-constructor.js TypeInputPeer
node <skill-dir>/scripts/get-constructor.js --with-references messages.sendMessage
```

`get-constructor.js` prints TL definitions, generated TypeScript types, union membership, return types, and optionally referenced constructors/unions.

## TypeDoc URL Patterns

Use hosted reference pages when a link is needed:

- Module: `https://ref.mtcute.dev/modules/_mtcute_node`
- Dispatcher filters: `https://ref.mtcute.dev/modules/_mtcute_dispatcher.filters`
- Class: `https://ref.mtcute.dev/classes/_mtcute_core.index.Message`
- Function/method exports vary by package and module; search the reference or use the scripts first.

The reference docs are generated by TypeDoc and show the package, source location, parameters, return type, and inherited members.

## Raw TL Calls

Use `.call()` for unwrapped Telegram API methods:

```ts
const result = await tg.call({
  _: 'account.checkUsername',
  username: 'example',
})
```

Rules:

- Use `get-constructor.js --with-references <method>` or `node_modules/@mtcute/core/tl/index.d.ts` for exact request shapes.
- Use `randomLong()` from `@mtcute/core/utils.js` when a raw method needs `randomId`.
- Resolve peers with `resolvePeer`, `resolveUser`, or `resolveChannel`.
- Convert resolved peers with `toInputUser` or `toInputChannel` when the raw method requires those narrower types.
- If a raw method returns `Updates`, call `tg.handleClientUpdate(result)` unless intentionally suppressing dispatch with `tg.handleClientUpdate(result, false)`.
- For affected-history style results with PTS, create a dummy update with `createDummyUpdate(...)`.
- Normalize raw files/media/text with `tg._normalizeInputFile`, `tg._normalizeInputMedia`, and `tg._normalizeInputText` only when high-level helpers are insufficient.

## TL Naming

Raw TL objects use `_` as a discriminant:

```ts
const raw: tl.RawUser = { _: 'user', id: 123, /* ... */ }
```

Common mappings:

- TL class `user` -> `tl.RawUser`
- TL namespaced class `messages.chats` -> `tl.messages.RawChats`
- TL method `users.getUsers` -> `tl.users.RawGetUsersRequest`
- TL union `User` -> `tl.TypeUser`

Optional flag fields are represented in generated types. Always consult generated typings before hand-writing raw calls.
